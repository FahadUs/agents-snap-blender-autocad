# Lessons for autocad_006 (AutoLISP & 3D / Automation) — read before every job

- [L006-x] On AutoCAD FULL, drive live work via COM/ActiveX (win32com): `GetActiveObject("AutoCAD.Application")` → `doc.ModelSpace.AddLine/AddCircle/AddArc/AddText/AddMText`. Points = `VARIANT(VT_ARRAY|VT_R8,[x,y,z])`; AddArc angles are in RADIANS. (source: live full-AutoCAD control proven via COM, File-IPC keystrokes failed) [cross-ref: autocad_001 L006]
- [COM] `doc.SendCommand` rejects ESC (\x1b) chars → "Invalid input"; send plain LISP/command strings ending with "\n". (source: ESC prefix error)
- [L017] COM dynamic dispatch can intermittently lose methods (e.g. AddLightWeightPolyline) when many drawings are open — wrap ModelSpace calls with a retry/re-fetch of `doc.ModelSpace`; avoid leaving many drawings open. (source: AttributeError, succeeded on retry)
- [L018] 2D→3D: set each closed wall polyline `.Thickness = ceiling_height` for quick 3D walls; iso view via SendCommand "_-VPOINT\\n1,-1,1\\n" + "_ZOOM\\n_E\\n" + "_VSCURRENT\\n_Conceptual\\n". For true SOLID walls: AddRegion(contours) → Boolean subtract room voids → AddExtrudedSolid(region, height). (source: 3D conversion)
- [L019] Heavy region-Boolean + MANY open drawings destabilises AutoCAD COM ("RPC server unavailable" / crash + Drawing Recovery). Mitigation: close extra drawings (Documents.Item(0).Close(False)) before heavy ops; prefer the lighter polyline.Thickness for walls + AddBox for floor; wrap COM calls in retry; save often. AddRegion takes a plain python list [pl] (NOT a VT_DISPATCH VARIANT, which throws "Invalid input"). (source: modern-3D boolean crash, recovered)
- [MODERN] Modern look = grey walls (color 9) on a warm floor slab (AddBox, color 43), Realistic visual style ("_VSCURRENT\\n_Realistic\\n"). (source: apartment_modern3d)
- [L020] Auto room segmentation via connectedComponents on (interior AND NOT dilated-walls) FAILS when door gaps are wide — rooms stay connected (got 2 comps). For furniture placement, MANUAL per-room centers (pixel→CAD) are more reliable than auto-segmentation. (source: furnished v2 nearly empty)
- [FURNISH] Furnished dollhouse in AutoCAD: white walls (255) + wood floor (43) + furniture as AddBox solids (bed/sofa/kitchen/dining/tub/wardrobe) at manual room centers; enable sun+sky ((setvar "SUNSTATUS" 1)(setvar "SKYSTATUS" 2)) + Realistic for render-like shadows. True photoreal → use Blender, not AutoCAD. (source: apartment_furnished3)
- [L025] 3D-via-COM view/shade safely: (a) set iso view by COM property — `vp=doc.ActiveViewport; vp.direction=VARIANT(...,(1,-1,1)); doc.ActiveViewport=vp; app.ZoomExtents()` (NO SendCommand, never hangs). (b) shading needs SendCommand `"_.VSCURRENT\n_C\n"` — `SetVariable("VSCURRENT",...)` throws "Error setting system variable". (c) ALWAYS SaveAs the model BEFORE any view/SendCommand (a hung view macro must never lose the build); (d) keep view+screenshot in a SEPARATE short script. Avoid multi-line `_.-VPOINT\n1,-1,1\n` — it tends to hang. (source: villa 3D build hung on VPOINT after save)
- [L027] Realistic lighting via COM: `SUNSTATUS=1` + `DEFAULTLIGHTING=0` is safe. AVOID `SKYSTATUS` in an isometric/parallel view — it requires Perspective ("Sky background can only be turned on in Perspective View") and opens the "Render Environment & Exposure" PALETTE which then BLOCKS subsequent COM/SendCommand (busy). If you need sky, switch to perspective first; otherwise sun-only. Apply Realistic via `SendCommand("_.VSCURRENT\n_R\n")`; close stray panels with `RENDERENVIRONMENTCLOSE`/`PROPERTIESCLOSE`/`CLEANSCREENON`. AutoCAD viewport ≠ photoreal — for true photoreal use Blender. (source: villa SKYSTATUS opened Exposure panel & hung the run)
- [L026] If COM sub-calls (Documents.Count/Add, ModelSpace) throw `AttributeError: <unknown>.X` while `app.Name` still works → AutoCAD is BUSY with a pending command (often from a prior hung SendCommand), not a dispatch bug. Recover by sending ESC keystrokes to the window: `win32gui.SetForegroundWindow(app.HWND)` then `win32api.keybd_event(0x1B,0,0,0)`/keyup ×2-3, wait, retry. (ESC via keybd works; ESC via SendCommand does not — see [COM] above.) (source: villa 3D, Documents.Add AttributeError cleared by ESC)
